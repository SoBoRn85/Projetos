# /i18n - Internationalization

$ARGUMENTS

---

## Purpose

This command activates internationalization (i18n) setup and localization (L10n) management using the `i18n-localization` skill. Use when adding multi-language support, auditing for hardcoded strings, or adding a new language.

---

## Sub-commands

```
/i18n                    - Audit current project for i18n issues
/i18n setup <language>   - Set up i18n library for the project stack
/i18n add <locale>       - Add a new language (e.g., /i18n add pt-BR)
/i18n scan               - Scan for hardcoded strings
/i18n rtl                - Add RTL language support
/i18n check              - Validate existing translations completeness
```

---

## Behavior

When `/i18n` is triggered:

### Step 1: Context Discovery

```bash
# Detect hardcoded strings and missing translations
python .agent/skills/i18n-localization/scripts/i18n_checker.py .
```

Identify:
- Current tech stack (React, Next.js, Vue, Python, etc.)
- Existing i18n setup (if any)
- Target languages requested

### Step 2: Select Implementation Pattern

| Stack | Library | Pattern |
|-------|---------|---------|
| React | react-i18next | `useTranslation()` hook |
| Next.js | next-intl | `useTranslations()` with routing |
| Vue | vue-i18n | `$t()` global |
| Python | gettext | `_()` wrapper |

### Step 3: Recommended File Structure

```
locales/
├── en/
│   ├── common.json      (shared strings)
│   ├── auth.json        (authentication)
│   └── errors.json      (error messages)
├── pt-BR/
│   ├── common.json
│   ├── auth.json
│   └── errors.json
└── ar/                  (RTL language)
    └── ...
```

### Step 4: Migration Checklist

- [ ] Run i18n_checker.py to find all hardcoded strings
- [ ] Create locale file structure
- [ ] Replace hardcoded strings with translation keys
- [ ] Set up date/number formatting with `Intl` API
- [ ] Test RTL layout (if applicable)
- [ ] Configure fallback language
- [ ] Update build/bundler config

/* Use CSS Logical Properties for RTL compatibility */

  margin-inline-start: 1rem;  /* NOT margin-left */
  padding-inline-end: 1rem;   /* NOT padding-right */
}

  transform: scaleX(-1);

## Key Principles

- **Plan RTL from day one** - retrofitting is painful
- **Never concatenate translated strings** - use interpolation
- **Allow 30% more space** - German/Portuguese are longer than English
- **Use ICU message format** - for complex pluralization
- **Test with real content** - not Lorem Ipsum
